Detailed Description

LibCRCx - The C API.

Typical use cases of libcrcx would be something like the following:

#include <crcx/crcx.h>
int main() {
const char data[] = { 'W' };
struct crcx_ctx ctx = {};
// Initialize an 8-bit CRC with no initializer or finalizer, where
// neither the input or output is reflected (in that order).
crcx_init(&ctx, 8, 0, 0, 0x07, false, false);
crcx(&ctx, data, sizeof(data));
uintmax_t crc = crcx_fini(&ctx);
// The value of crc should be 0xa2.
return 0;
}

Compute the CRC.

This function should be called after crcx_init and before crcx_fini.

It validates input with crcx_valid and then calls crcx_update len times, once for each item in data.

Typical uses cases of CRCx will use this function along with crcx_init and crcx_fini. However, doing so is simply a shorthand. It is entirely possible to call crcx_update directly without the use of this function.

Parameters
ctxthe CRC context to use
datathe data for which the CRC should be calculated
lenthe length of data
Returns
true on success, otherwise false

Finalize a CRC context.

This function xor's the crcx_ctx::fini value with the crcx_ctx::lfsr.

It also reflects the crcx_ctx::lfsr if crcx_ctx::reflect_output is set.
